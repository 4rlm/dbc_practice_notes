    3.4 Mandatory Pairing: Responsive Design

    Submit Assignment
    • Due Jan 3, 2017 by 10am
    • Points 30
    • Submitting a website url

    These days, screens come in all shapes and sizes: laptops, smartphones, tablets, and so on. Some users will view your site on their wrists, using a screen the size of a postage stamp. Users on the other end of the spectrum have cleared their entire desks to fit a monitor approximately the size of Texas.

    "Please make your website look good on both of these screens ... along with my tablet and my giant gaming monitor that can be seen from space."

    Responsive design refers to a collection of techniques that adjust the appearance of a site depending on the width of the screen. If you want to define the term more loosely, responsive design might also be thought of as building a site that responds to its environment. Broader topics in responsive design might address questions like

    • How can we make sure to accommodate both touchscreens and mice?
    • Does this site look good in older browsers?
    • Are mobile users accomplishing the same set of tasks as laptop users, or are the goals of a mobile user different? Should the menu options and buttons change depending on the platform in order to reflect these different priorities?

    In this challenge, we'll introduce you to a few strategies of basic responsive design.

    Release 0: Build a Simple Layout

    do the thing
    1. We left you a sample project in phase-0-tracks/css/responsive-design. Take a look at what's already there, and open index.html in your browser.
    2. Include the provided stylesheet in index.html.
    3. The entire page is pretty wide, which is hard to read. Use CSS to set the width of the body element to 960 pixels.
    4. The External Links section should be in a sidebar. Using its ID as a selector, set its width to 200px, and set the width of the main content to 756px (these numbers don't quite add up to 960px due to a quirk of the display property). Use the display and vertical-align properties to put the External Links section side-by-side with the main content.

    Okay, the full-width version of the site is looking a little better. Now we need to make it responsive.

    Release 1: Check the Responsiveness

    You can check the responsiveness of your page so far using Chrome Dev Tools.

    Resizing with Dev Tools

    If you resize the viewport (the visible area) of the site with Dev Tools, the width of the viewport will be displayed, so you can take note of any width ranges that look odd:

    (The option to dock your Dev Tools on the side of the browser window instead of at the bottom can be found by clicking the three dots right next to the X you'd click to exit Dev Tools.)

    Previewing the Site on Mobile Devices

    Dev Tools offer previews of various mobile devices. Just click the smartphone icon, then choose the device and the orientation (e.g., portrait or landscape). Notice that sometimes you need to refresh the page for reliable results.

    do the thing
    1. Use Dev Tools to preview the responsiveness of the responsive-design site at various screen widths and on a few devices. Does the text ever get cut off or look very small?
    2. Instead of setting the width of the body tag, set the max-width to 960px instead. max-width is more flexible.
    3. Instead of using pixels for your sidebar and main widths, use percentages (example: width: 20%;) to create a similar look with more flexibility. Using fluid widths like this will make the site more responsive.
    4. Test the responsiveness again using Dev Tools, and make note of any widths that still look odd. Do you find you need to add padding to avoid near-crashes of text, or min-width declarations to keep lines from getting too short? Tweak these properly as needed.
    5. Do very narrow widths still look odd? We'll fix those in the next release.

    Even our simple adjustments, like using fluid widths and maximum widths, have helped a lot, but there's more we can do to accomplish the exact effect we want.

    Release 2: Add Media Queries

    A media query allows you to apply CSS conditionally based on a variety of attributes. If we use a media query to apply CSS for certain screen widths, we can make our page look better at narrow widths.

    try the thing

    Drop the following CSS into the bottom of style.css:

    @media screen and (max-width: 500px) {
  body {
    background-color: hotpink;
  }
}

    This is a media query. What do you think it does? Make a guess, then check your guess by dragging the screen width back and forth across the 500px mark.

    While this particular media query isn't of much use, you'll probably want to comment it out and keep it as a syntax reference.

    Media queries can be used for all sorts of responsive design decisions, including the following:

    • Making heading or other text smaller at very narrow widths
    • Stacking page sections on top of one another and setting their widths to 100%
    • Stacking images on top of one another at narrow widths
    • Adding a third or even a fourth column of additional content at very wide widths
    • Switching menu options from words to icons as widths shrink

    As you drag a site and reduce the width of its viewport, you can make note of any widths that look odd, then write media queries at that width to adjust the appearance. This width becomes a breakpoint, or a place where the design of the site changes.

    do the thing

    Add at least two media queries to the responsive-css project that improve its appearance. The first two ideas in the list above will work, or feel free to be creative. (We just ask that the queries are used to improve the responsiveness of the site, rather than accomplishing something random like a pink background.)

    Release 3: Research with Your Pair

    find the thing
    1. How can you center the content of the entire page instead of having it sit to the left? Try it in the responsive-css site if you figure it out in time.
    2. What is mobile-first design?
    3. What is the float property in CSS? What are the pros and cons of using the float property instead of the display property?

    To complete this challenge, submit the GitHub URL of your CSS file.

    Release 4: Give Feedback to Your Pair

    do the thing

    Take a moment to give each other actionable, specific, and kind feedback on your pairing session. An example of actionable, specific, and kind feedback might be, "Sometimes we went faster than I was comfortable with because you were on a roll. It might help to pause here and there to make sure your pair is still with you. But you were really encouraging, and I appreciated your positive attitude."

    By now, exchanging ASK feedback should be part of your pairing routine. We won't remind you on every pairing challenge from here on out, so make sure you remind each other!
